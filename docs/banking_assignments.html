30-Day Banking Domain Assignments

Each assignment maps to the core curriculum you provided (Core Java through Spring/React stacks, Git, and testing) minus infrastructure. All relate to Broadridge-style workflows (accounts, transactions, compliance, scorecards).

Assignment Instructions

Deliver each assignment as its own Git branch with descriptive commit(s), unit tests where applicable, and a short README summarizing the business rule and how to exercise the feature.

  1. Account Onboarding CLI

    Build a CLI that registers retail clients with `Customer`, `PrimaryAccount`, and `Address` classes. Persist data to an in-memory map and log every created account.

    CollectionsObject ModelingCLI

  2. Transaction Balance Checker

    Create a service that stores transactions in a `List` and throws `InsufficientFundsException` when debit exceeds balance. Include unit tests showing the exception path.

    Exception HandlingJUnit

  3. Customer Statement Generator

    Design an immutable `StatementLine` record. Build a `StatementService` that sorts lines by timestamp, groups by transaction type, and formats totals.

    CollectionsSortingJava 17

  4. Git-Based Code Review Workflow

    Document in Markdown how to create feature branches, commit messages referencing Jira-style IDs, and open a pull request for code reviewing a banking feature. Include `git` commands and templates.

    GitGitHubDocumentation

  5. Account Authorization Interface

    Define a `BankOperation` interface with methods `authorize()` and `execute()`. Implement `FundTransferOperation` and `BillPaymentOperation`, then register them with a `BatchProcessor`.

    InterfacesPolymorphism

  6. Exception Observation Dashboard

    Capture exceptions during transaction processing, categorize by type (checked/unchecked), and log metadata. Provide a simple HTML report listing counts per exception class.

    Exception HandlingCollections

  7. Generics-Based Cache

    Implement a thread-safe `GenericCache` backed by `ConcurrentHashMap`. Require `V` to extend `Cacheable` and provide `putIfAbsent`/`computeIfPresent` helpers.

    GenericsConcurrency

  8. Props & Config Loader

    Load `application.properties` with JDBC credentials and API keys. Provide defaults, allow overrides via environment variables, and surface values through a `Config` singleton.

    PropertiesSingleton

  9. SQL Schema for Ledger

    Design `accounts`, `ledgers`, and `transactions` tables with PK/FK constraints. Share the DDL script plus sample INSERT statements demonstrating constraints.

    DDLIntegritySQL

  10. JOIN-Based Balance Report

    Write SQL that joins accounts with transactions to compute daily balances, highlighting overdrafts. Include a view definition or stored query for analytics.

    Advanced SQLReporting

  11. PL/SQL Alert Procedure

    Implement a PL/SQL procedure that scans the ledger, detects suspicious `amount > 10000` transfers, and inserts rows into an `alerts` table.

    PL/SQLTriggers

  12. JPA Entities & Relationships

    Model `Customer`, `Account`, and `Transaction` entities with `OneToMany`/`ManyToOne` relationships. Seed data using `CommandLineRunner`.

    JPASpring Boot

  13. Spring Boot REST API

    Expose endpoints for `/accounts`, `/transactions`, `/alerts`. Use DTOs and validation annotations. Return meaningful error responses for validation or not-found cases.

    Spring MVCValidation

  14. Spring Data JPA Queries

    Implement repository methods using method names and `@Query` to find high-value transactions and get account balances via projections.

    Spring DataRepositories

  15. Spring Security Layer

    Secure APIs with HTTP Basic or JWT, restrict transaction creation to `ROLE_TELLER`, and return `403` for unauthorized attempts. Provide test coverage for secured endpoints.

    SecurityRoles

  16. Frontend React Banking Dashboard

    Build a React app that hits `/accounts` and `/transactions`, displays balances, and highlights alerts. Use hooks for state and axios for REST calls.

    ReactHooks

  17. React Form & Validation

    Create a transfer form with React Router, validate fields (amount > 0, account numbers required) and show inline error messages before submitting.

    FormsValidation

  18. Redux Toolkit State

    Use Redux Toolkit slices to store accounts + alerts. Connect a component to display live alert counts.

    ReduxState

  19. E2E Story Flow

    Describe an end-to-end scenario: onboarding, transfer, alert creation, REST call, and React UI update. Provide sequence diagrams or textual steps.

    IntegrationStory

  20. JUnit 5 Test Suite

    Write unit tests for the service layer covering success, exception, and boundary cases. Use `@Nested`, `@DisplayName`, and parameterized tests for coverage.

    JUnit 5TDD

  21. GitHub Issue Tracker

    Create issue templates for `bug`, `feature`, and `research`. Link them to sample assignments and show a workflow that ties issues to commits.

    GitHubTemplates
Students should submit each assignment with a short demo video or README showing how the banking scenario works plus any relevant SQL scripts/tests.
