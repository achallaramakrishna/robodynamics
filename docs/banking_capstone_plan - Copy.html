Database Structure

  • accounts (account_id PK, customer_id FK, account_type, balance, status, created_at)
  • customers (customer_id PK, first_name, last_name, email UNIQUE, created_at)
  • transactions (transaction_id PK, account_id FK, type, amount, timestamp, status, description)
  • alerts (alert_id PK, transaction_id FK, level, message, created_at)
  • ledgers (ledger_id PK, account_id FK, entry_type, amount, recorded_at)
  • Indexes on `customers.email`, `accounts.customer_id`, `transactions.account_id`, and composite indexes for reporting queries.
  • PL/SQL procedure `raise_high_value_alerts` runs nightly, scanning large transfers and inserting into `alerts`.

Submission Guidelines

  • Each team consolidates work into a shared branch, includes SQL scripts/tests, and records a short demo.
  • Reviews evaluate correctness, tests, security, and integration with other teams.
