15-Phase Banking Project Plan

Sequential phases guide interns from Core Java modeling through SQL/PLSQL, Spring/Hibernate APIs, Git practices, and React/Node integration to deliver a complete banking application.

  1. Domain Modeling

    Build `Customer`, `Account`, `Transaction`, and `StatementLine` classes/records with validation helpers and overrides so future layers reuse consistent structures.

  2. Business Rule Enforcement

    Create services (e.g., `TransactionService`) that throw `InsufficientFundsException` when debits exceed balance and cover the rules with JUnit 5 tests.

  3. Reporting Logic

    Implement a `StatementService` that sorts transactions, groups by type, and formats totals using comparators and collection pipelines.

  4. Git Process Documentation

    Document branching conventions, commit structure, PR checklist, and GitHub issue templates tailored to the banking workflow.

  5. Operation Interface Layer

    Introduce a `BankOperation` interface plus implementations (`FundTransfer`, `BillPayment`), then register them with a batch processor to exercise polymorphism.

  6. Exception Monitoring

    Capture exceptions thrown by service methods, tally them by type, and expose counts via an HTML report for visibility before the Spring layer integrates it.

  7. Configuration Loader

    Build a singleton that loads `application.properties`, honors environment overrides, and exposes typed getters (JDBC URL, API secrets).

  8. Ledger Schema (DDL)

    Write SQL for `accounts`, `transactions`, `ledgers`, and `alerts` tables with PK/FK constraints, indexes, and seed inserts representing the domain.

  9. Analytical SQL Queries

    Produce JOIN-based views/reports that compute daily balances and highlight overdrafts for reporting dashboards.

  10. PL/SQL Alert Procedure

    Implement a PL/SQL stored procedure that scans transfers >10k and populates an `alerts` table for compliance monitoring.

  11. JPA Entities & Relationships

    Map the schema to Spring entities with proper relationships and seed initial data via `CommandLineRunner` for the REST API.

  12. Spring REST API

    Expose `/accounts`, `/transactions`, `/alerts` controllers using DTOs, validation annotations, and structured error handling tied to the exception layer.

  13. Spring Data + Security

    Add repository methods (`@Query`) for high-value transactions and balances, then secure endpoints so only `ROLE_TELLER` can create transactions.

  14. React Frontend

    Build a React dashboard that consumes the Spring API, shows balances/alerts, and includes a transfer form with validation using hooks.

  15. Redux & Integration Story

    Manage accounts/alerts via Redux Toolkit slices, connect components for live alert counts, and document the end-to-end flow from onboarding through alert generation.

Each phase must include README notes, tests or SQL scripts, demo instructions, and be delivered via Git branches following the documented workflow. Together they form a production-like banking application.
