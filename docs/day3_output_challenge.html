Output Prediction Challenges

Give these puzzles in small groups. Let teams debate the output before showing the answer and dissecting the behavior.

Challenge 1: Order of finally

Duration: 5 minutes | Concept: try/catch/finally precedence

public static String flag() {
  try {
    return \"try\";
  } finally {
    return \"finally\";
  }
}

System.out.println(flag());

What prints to the console?

  • A) try
  • B) finally
  • C) tryfinally
  • D) compilation error

Hint: finally executes after try, even if a return already executed.

Challenge 2: Iterator remove

Duration: 7 minutes | Concept: Iterator contract and ConcurrentModificationException

List<String> names = new ArrayList<>(List.of(\"Alice\", \"Bob\", \"Cara\"));
for (String name : names) {
  if (name.startsWith(\"C\")) {
    names.remove(name);
  }
}
System.out.println(names);

What happens when this code runs?

  • A) [Alice, Bob]
  • B) [Alice, Cara]
  • C) ConcurrentModificationException
  • D) Compilation error

Hint: fail-fast iterators disallow modifying the underlying collection while looping.

Challenge 3: Generics with

Duration: 6 minutes | Concept: Wildcards and type capture

List<? extends Number> values = List.of(1, 2, 3);
values.add(4);
System.out.println(values);

What is the result?

  • A) [1, 2, 3, 4]
  • B) Compilation error at add()
  • C) UnsupportedOperationException
  • D) RuntimeClassCastException

Hint: ? extends Number prevents adding because the compiler loses track of the exact subtype.

Reveal the answers together, then describe the lifecycle for try/catch/finally, iterator modification, and PECS.
