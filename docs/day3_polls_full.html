Classes (5 Polls)

Cover constructors, encapsulation, and class responsibilities before switching to examples.

1. What keyword creates a new object?

  • new
  • class
  • public
  • extends

Hint: Instantiation always calls constructors using this keyword.

2. Which visibility modifier is package-private?

  • private
  • protected
  • (no modifier)
  • final

Answer: Omitting a modifier keeps members visible inside the same package only.

3. Which method expresses an object's state for logging?

  • toString()
  • equals()
  • hashCode()
  • notify()

toString() is a logging-friendly describe method every value object should override.

4. How do you prevent subclassing?

  • abstract
  • static
  • private
  • final

Make the class final so the compiler rejects any extends attempts.

5. What best describes a model class?

  • Holds data only
  • Contains business logic
  • Manages threads
  • Handles I/O

Correct: Model classes focus on representing structured data, not behavior.

Inheritance (5 Polls)

Confirm understanding of extends, method overriding, and the is-a relationship.

1. Which keyword calls the superclass constructor?

  • this
  • super
  • extends
  • base

Hint: super() jumps up one level.

2. What happens if you override without @Override?

  • Compilation error
  • It still overrides
  • Runtime warning
  • Method is final

Answer: It still overrides, but @Override helps catch typos.

3. How to call parent logic from an override?

  • super.method()
  • parent.method()
  • this.method()
  • base.method()

Use super.method(...) to reuse behavior before extending it.

4. Which describes inheritance?

  • Has-a
  • Uses-a
  • Is-a
  • Needs-a

Car is a Vehicle is a classic inheritance example.

5. When should you use inheritance?

  • Different animals sharing eat()
  • Different UIs for separate apps
  • Networking vs disk I/O
  • Bank vs Insurance services

Pick a domain with shared behavior and a natural parent entity.

Interfaces (5 Polls)

Validate awareness of default methods, multiple interfaces, and conceptual differences.

1. Which keyword implements an interface?

  • extends
  • implements
  • uses
  • super

implements binds a class to an interface contract.

2. Which feature arrived in Java 8 interfaces?

  • Can hold state
  • Can have default methods
  • Can be instantiated
  • Members are private

Default methods ship reusable logic alongside abstract declarations.

3. Can a class implement multiple interfaces?

  • Yes
  • No
  • Only if abstract
  • Only with generics

Yes—interfaces are the route around Java's single inheritance.

4. Why prefer interfaces over abstract classes?

  • Avoid multiple inheritance limitations
  • Require constructors
  • Hold business data
  • Provide state

Interfaces allow behavior to be mixed into many classes.

5. Which belongs on an interface?

  • start() and stop()
  • Static factory
  • Private fields
  • Final methods

Behavioral contracts such as start/stop suit interfaces best.

Check comprehension of static context, fields, and utility usage.

1. Static belongs to?

  • Class

  • Interface

2. When should constants be static final?

  • Only for methods
  • For shared, immutable values
  • For constructors
  • Never

Share static final values instead of hard-coding per object.

3. Which block runs once when a class loads?

  • Instance initializer
  • Static initializer
  • Constructor
  • Finalizer

Static initializer blocks are ideal for bootstrapping shared resources.

4. Can an interface define static members?

  • Yes
  • No
  • Only in abstract classes
  • Only with enums

Interfaces can have static helpers since Java 8.

5. Why use static factory methods?

  • To reuse constructors
  • To avoid new keywords
  • Every class should have one
  • When constructors are private

Static factories hide construction details and can return cached instances.

Exception Handling (5 Polls)

Reinforce which keywords control try/catch flow and how checked/unchecked differ.

1. Which is checked even if not declared?

  • IOException
  • RuntimeException
  • Error
  • NullPointerException

IOException requires throws or handling.

2. What guarantees execution even with a return?

  • try
  • catch
  • finally
  • throw

finally runs regardless of earlier returns.

3. Which handles multiple exception types elegantly?

  • multi-catch
  • finally
  • throw
  • try-with-resources

catch (SQLException | IOException e) is multi-catch.

4. What's needed to use try-with-resources?

  • Resource implements AutoCloseable
  • Use try only
  • Call close() manually
  • Wrap in finally

Any resource that implements AutoCloseable works.

5. When throw vs throws?

  • throw declares exceptions
  • throws instantiates them
  • throw raises an exception at runtime
  • throws runs code in catch

throw triggers an exception explicitly, throws lists it on the method.

Quiz learners on interfaces, implementations, and iteration strategies.

1. Which interface is sorted?

  • Set
  • Queue
  • Map

Many implementations maintain sorting indirectly through comparators.

2. Which Set keeps insertion order?

  • HashSet
  • LinkedHashSet
  • TreeSet

3. How does HashMap handle collisions?

  • Linked lists/buckets

4. Which iteration gives fail-fast behavior?

5. What utility class sorts?

Generics (5 Polls)

Make sure learners know bounded types, wildcards, and erasure limits.

1. Which declaration lets you accept Number subclasses?

  • <T>
  • <T extends Number>
  • <T super Number>
  • <? extends Number>

Use <T extends Number> when you need numeric API access.

2. PECS stands for?

  • Producer Extends Consumer Super
  • Pattern Extend Class Strategy
  • Proxy Extract Class Super
  • Plain Enum Control Style

Remember: Producer Extends, Consumer Super.

3. Can you instantiate new T()?

  • Yes
  • No
  • Only with reflection
  • Only if T extends Object

You cannot create new T() directly because of type erasure.

4. Which wildcard lets you add integers safely?

  • List<?>
  • List<? extends Integer>
  • List<? super Integer>
  • List<T>

Use ? super Integer when adding values.

5. Wildcard ? extends T allows:

  • Read only
  • Write only
  • Both read and write
  • No access

Producer Extends: you can read, but adding is limited.
