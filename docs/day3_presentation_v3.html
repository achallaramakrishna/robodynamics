Exception Handling

Slide 3 · Try / catch / finally anatomy

Cover the runtime journey of an exception, mention the difference between checked (IOException) and unchecked (NullPointerException) types, and stress when to rethrow with context.

try (BufferedReader reader = Files.newBufferedReader(path)) { String line = reader.readLine(); System.out.println(line); } catch (IOException e) { log.error("Cannot read config", e); } finally { System.out.println("Cleanup log written"); }

Highlight: finally runs even when you return early; multi-catch lets you handle similar exceptions without duplication.

Debug Challenge

Slide 4 · Predict the output

Click into this code before showing the execution.

try { System.out.println("A"); throw new IllegalStateException("oops"); } catch (RuntimeException e) { System.out.println("B"); } finally { System.out.println("C"); }

Poll

When should you log an exception?

A Every time you catch, regardless of context.
B Only when you rethrow with more context.
C When the exception crosses module/API boundaries and someone needs to know.
D Never log; rely on upstream handlers.

Correct: C. Log at boundaries with context; avoid double logging that floods the console.

Collections · Overview

Slide 8 · Choosing the right Collection

  • List: Ordered, accepts duplicates; use ArrayList for random access, LinkedList for insert-heavy flows.
  • Set: Uniqueness. HashSet for speed, TreeSet for sorted order, LinkedHashSet to preserve insertion order.
  • Map: Key-value store; HashMap vs TreeMap (natural ordering) vs LinkedHashMap (recency/insertion), ConcurrentHashMap for multi-threaded access.

Example

Generics · Type safety

Slide 11 · Wildcards & PECS

  • Producer extends: List<? extends Number> → can read Number but not add.
  • Consumer super: List<? super Integer> → can add Integer but reading only guarantees Object.
  • Generics prevent runtime ClassCastException by enforcing types at compile time.
public static double sumNumbers(List<? extends Number> numbers) { return numbers.stream() .mapToDouble(Number::doubleValue) .sum(); }
