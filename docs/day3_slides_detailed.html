Slide 2

Exception Handling Basics

Time: 1:10 – 1:30 PM

Concepts you cover:

  • Try/catch/finally lifecycle, difference between caught vs auto-propagated errors.
  • Checked vs unchecked exceptions and why the compiler enforces certain contracts.
  • When to use throw vs throws.
try (BufferedReader reader = Files.newBufferedReader(path)) { String line = reader.readLine(); // ... } catch (IOException e) { log.error("Unable to read config", e); } finally { System.out.println("This always executes unless System.exit"); }

Slide Activity

Start with a crash reproduction: run a demo that throws FileNotFoundException and show the stack trace. Ask students to annotate each line as “handled” or “bubbled up”.

Slide 3

Debug + Output Challenge

Live debugging prompt

  • Inject a NullPointerException by calling String.toUpperCase() on a null value inside a try block.
  • Step through the call stack with debugger or print statements; ask: “Which stack frame belongs to the catch block?”

Output Prediction

Show the following snippet and ask attendees to predict the printed lines before running.

try { System.out.println("Step A"); throw new IllegalArgumentException("Bad input"); } catch (RuntimeException e) { System.out.println("Step B"); } finally { System.out.println("Step C"); }

Answer: Step A → Step B → Step C (finally always executes even after catch).

Slide 4

Poll: Exception Logging Strategy

"When should you log an exception?"

  • A) Only if it’s not rethrown.
  • B) In every catch block.
  • C) When the exception crosses module/API boundary and there is no upper-level handler.
  • D) Never log; let the caller handle it.

Correct answer: C – log at boundaries or when you have actionable context, but avoid double logging upstream.

Slide 6

Collections Framework – Deep Dive

Time: 2:40 – 3:20 PM

List / Set / Map fundamentals

  • List: ordered and indexed. Compare ArrayList vs LinkedList (array vs node, random access vs insertion).
  • Set: uniqueness. Distinguish HashSet (unordered), LinkedHashSet (insertion order), TreeSet (sorted).
  • Map: hash-based vs tree-based; show how HashMap handles collisions, TreeMap provides natural ordering.

Example

Maintain a playlist with List<Song>, then extract unique artists via Set<String> + Map<String, Integer> for play counts.

Best Practice

Prefer interfaces in signatures: List<String> instead of concrete type, and choose the smallest capability you need (e.g., Deque when you need push/pop).

Slide 7

Collections Advanced – Concurrency & Streams

  • Trigger ConcurrentModificationException via for-each removal; fix with Iterator.remove() or use CopyOnWriteArrayList.
  • Introduce ConcurrentHashMap (segment-based locking) and describe when to use it vs synchronized map.
  • Stream helper: students.stream().filter(...).collect(Collectors.toList()) to create immutable snapshots.

Programming Exercise

Slide 8

Generics Essentials – Type Safety

Time: 4:00 – 4:30 PM

  • Explain generics invariance, covariance vs contravariance via PECS (Producer Extends, Consumer Super).
  • Show signature: <T extends Comparable<T>> T findMax(List<T> list) and why comparables are required.
  • Highlight raw type pitfalls: unchecked casts and ClassCastException at runtime.
public static void addIntegers(List<? super Integer> list) { list.add(1); list.add(2); }
Slide 9

Generics Practice Lab

  • Refactor the lab method that currently uses raw lists of Object into typed helpers (List<String>, List<Number>).
  • Implement helper: public static double sumNumbers(List<? extends Number> numbers) and explain compile-time limitations.

PECS Reminder

Use ? extends when you only read elements, ? super when you only write.

Debug Challenge

Show a method that tries to add an Integer to a List<? extends Number> and explain the compiler error.
