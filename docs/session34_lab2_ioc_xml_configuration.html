Lab Manual 2

Implementing IoC using XML Configuration

📘 Objective

In this lab, students will learn how Spring removes tight coupling by managing object creation using the IoC container through XML configuration.

🧠 Concept: What is IoC?

Traditional Java:

MessageService service = new EmailService();

The application controls object creation.

Spring IoC:

Spring container creates and manages objects (Beans). Your application only requests them.

Control of object creation is inverted → Hence Inversion of Control.

Step 1 – Add Spring Dependency (pom.xml)

<dependency> <groupId>org.springframework</groupId> <artifactId>spring-context</artifactId> <version>5.3.30</version> </dependency>

After adding dependency → Right Click Project → Maven → Update Project

Step 2 – Create Interface

package com.robodynamics.springcore.service; public interface MessageService { String sendMessage(String to, String message); }

Step 3 – Create Implementation

package com.robodynamics.springcore.service; public class EmailService implements MessageService { @Override public String sendMessage(String to, String message) { return "EMAIL SENT via Spring IoC to " + to + " | message=" + message; } }

Step 4 – Create Spring XML Configuration

Create file under: src/main/resources/applicationContext.xml

<beans xmlns="http://www.springframework.org/schema/beans" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:schemaLocation=" http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd"> <bean id="messageService" class="com.robodynamics.springcore.service.EmailService"/> </beans>

Here Spring creates the object. We do NOT use new.

Step 5 – Create Main Class

package com.robodynamics.springcore; import org.springframework.context.ApplicationContext; import org.springframework.context.support.ClassPathXmlApplicationContext; import com.robodynamics.springcore.service.MessageService; public class App { public static void main(String[] args) { ApplicationContext context = new ClassPathXmlApplicationContext("applicationContext.xml"); MessageService service = context.getBean("messageService", MessageService.class); System.out.println( service.sendMessage("student@robodynamics.in", "Welcome to Spring IoC!") ); } }

Expected Output

EMAIL SENT via Spring IoC to student@robodynamics.in | message=Welcome to Spring IoC!

🔁 Execution Flow

  1. Spring container starts.
  2. Reads applicationContext.xml.
  3. Creates EmailService bean.
  4. Stores bean inside container.
  5. App requests bean using getBean().
  6. Spring returns managed object.

🚨 Common Errors & Fixes

  • Cannot find spring-context: Run Maven Update
  • applicationContext.xml not found: Ensure file inside resources
  • NoSuchBeanDefinitionException: Check bean id name
  • ClassNotFoundException: Verify fully qualified class name in XML

🧠 Reflection

We no longer use new EmailService(). Spring container creates and manages the object. This removes tight coupling.
